# 校园资产智慧巡检与全生命周期管理平台

实践周项目 01。项目根据课程 PPT 的校园工单系统要求扩展为“主动巡检 + 预防性维护”的资产管理系统。

## 启动

```bash
cd 01-campus-asset-inspection
node server.js
```

浏览器打开：

```text
http://localhost:3101
```

数据会保存到 `data/db.json`，刷新浏览器后不会丢失。

## 演示账号

进入首页后直接选择角色：

- 资产管理员：资产入库、受理异常、派单、重新受理
- 巡检员：执行巡检，异常时自动生成工单
- 维修人员：开始处理、提交维修结果
- 师生用户：确认完成或反馈未解决

## 答辩演示路径

1. 用“资产管理员”进入系统，在资产管理中新增资产。
2. 切换“巡检员”，选择资产并提交异常巡检。
3. 系统自动生成维修工单，资产状态变为“维修中”。
4. 切换“资产管理员”，受理并派单给维修人员。
5. 切换“维修人员”，开始处理并提交维修结果。
6. 切换“师生用户”，确认完成，资产状态恢复“正常”。
7. 打开日志通知页，说明状态、日志、通知的可追踪闭环。

## 对照课程要求

- 前端：单页后台应用，包含角色入口、路由式导航、表格、表单、看板和通知。
- 后端：Node 服务提供静态资源和 `/api/state` 持久化接口；`prisma/schema.prisma` 给出课程要求的 PostgreSQL/Prisma 数据模型。
- 角色：资产管理员、巡检员、维修人员、师生用户。
- 核心闭环：资产入库 → 巡检计划 → 执行巡检 → 异常工单 → 维修处理 → 用户确认 → 状态归档。
- 工程交付：可运行系统、初始化数据、README、Docker Compose 数据库配置、Prisma 模型。

## 课程栈说明

为了保证答辩现场稳定，本版本不依赖外网安装包，使用零依赖 Node 服务运行。目录中保留了可迁移到课程标准栈的 `prisma/schema.prisma` 与 `docker-compose.yml`；正式扩展时可将页面迁移到 Vue 3 + TypeScript + Antd Vue，将 `/api/state` 拆分为 NestJS 模块接口。
